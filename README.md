# SyncMaster

**SyncMaster** — CLI-утилита на Node.js/TypeScript для синхронизации сериалов и видеофайлов между исходными и целевыми папками с формированием текстового отчёта.

## Основной функционал

- **Синхронизация сериалов** (`series`):
    - Чтение настроек из `src/config/setting.json` (поля `enabled`, `name`, `src`, `dest`).
    - Маппинг исходных и целевых названий папок из `src/config/soap.csv` (разделитель `;`).
    - Сравнение содержимого папок: недостающие файлы копируются из `src` в `dest` и переименовываются по номеру эпизода (регулярка `\.s\d+\.?e(\d+)` или ведущие цифры в имени).
    - Файлы с расширением `.!qB` (незавершённые загрузки qBittorrent) пропускаются.
    - Исходные файлы **сохраняются** после копирования.

- **Синхронизация видео** (`editorial_video`, `turkish_video`):
    - Плоское сравнение папок `src` и `dest` без вложенных каталогов.
    - Имена файлов длиннее 50 символов обрезаются по последнему пробелу.
    - После успешного копирования исходный файл **удаляется**.

- **Создание отчёта:**
    - Формируется файл вида `report_DDMMYYYY_HHMM.txt` с отсортированными названиями скопированных файлов.
    - Отчёт создаётся только если хотя бы одна из синхронизаций скопировала файлы.
    - Путь к отчёту: `src/report/` в режиме разработки и `dist/report/` в продакшене.

- **Логирование:**
    - Winston пишет одновременно в консоль (цветной вывод) и в файл `logs/syncmaster.log`.

## Требования

- Node.js >= 20.0.0

## Настройка

1. Отредактируйте `src/config/setting.json`, задав пути `src` и `dest` для трёх блоков: `series`, `editorial_video`, `turkish_video`. Ненужные блоки можно отключить флагом `"enabled": false`.
2. Для блока `series` заполните `src/config/soap.csv` в формате `исходная_папка;целевая_папка`.
3. Убедитесь, что в `.env` задан `NODE_ENV` (`development` для запуска из исходников, `production` — для бандла).

## Команды

- `npm run dev` — запуск из исходников через `tsx` (режим разработки).
- `npm run build` — сборка в один файл `dist/index.js` через `@vercel/ncc`, копирование конфигов в `dist/config`.
- `npm run run` — сборка и запуск через `ncc run`.
- `npx eslint .` — проверка линтером.
- `npx prettier --write .` — форматирование.

После `npm run build` бандл можно запускать как `node dist/index.js`.
