# SyncMaster

**SyncMaster** — CLI-утилита на Node.js/TypeScript для синхронизации сериалов и видеофайлов между исходными и целевыми папками с формированием текстового отчёта.

## Основной функционал

- **Синхронизация сериалов** (`series`):
  - Чтение настроек из `settings.yml` по пути из переменной окружения `SETTINGS_PATH`.
  - Маппинг исходных и целевых названий папок из секции `series_map` в виде массива объектов `{ src, dest }`.
  - Сравнение содержимого папок: недостающие файлы копируются из `src` в `dest` и переименовываются по номеру эпизода (регулярка `\.s\d+\.?e(\d+)` или ведущие цифры в имени).
  - Файлы с расширением `.!qb` (незавершённые загрузки qBittorrent) пропускаются.
  - Исходные файлы **сохраняются** после копирования.

- **Синхронизация видео** (`editorial_video`):
  - Плоское сравнение папок `src` и `dest` без вложенных каталогов.
  - Имена файлов длиннее 50 символов обрезаются по последнему пробелу.
  - После успешного копирования исходный файл **удаляется**.

- **Создание отчёта:**
  - Формируется файл вида `report_DDMMYYYY_HHMM.txt` с отсортированными названиями скопированных файлов.
  - Отчёт создаётся только если хотя бы одна из синхронизаций скопировала файлы.
  - Путь к отчёту: `<ROOT_DIR>/report/`.

- **Логирование:**
  - Winston пишет одновременно в консоль (цветной вывод) и в ежедневно ротируемые файлы `logs/syncmaster-YYYY-MM-DD.log` (макс. 20 МБ, хранение 30 дней, gzip-архивация).
  - Директория логов: `LOG_DIR` из окружения (если задан) или `<ROOT_DIR>/logs/`.

- **TUI-редактор маппингов** (`menu-editor`):
  - Интерактивный редактор секции `series_map` в `settings.yml` на базе `@clack/prompts`.
  - Позволяет добавлять и удалять записи с опциональным удалением соответствующих папок.

## Требования

- Node.js >= 20.0.0

## Настройка

1. Задайте в `.env` переменную `SETTINGS_PATH`, например `SETTINGS_PATH=./settings.yml`.
2. Опционально задайте `LOG_DIR` для хранения логов в произвольном каталоге.
3. Отредактируйте `settings.yml`: задайте пути `src` и `dest` для блоков `series` и `editorial_video`. Ненужные блоки отключаются через `enabled: false`.
4. Для блока `series` заполните секцию `series_map` в формате массива объектов `{ src, dest }`.

## Команды

- `npm run dev` — запуск синхронизации из исходников через `tsx`.
- `npm run menu` — запуск TUI-редактора маппингов через `tsx`.
- `npm run build:sync` — сборка синхронизатора в `dist/index.js` через `@vercel/ncc`.
- `npm run build:menu` — сборка TUI-редактора в `dist/menu.js` через `@vercel/ncc`.
- `npm run build:all` — сборка обоих бандлов.
- `npx eslint .` — проверка линтером.
- `npx prettier --write .` — форматирование.

После сборки запускать как `node dist/index.js` и `node dist/menu.js` соответственно.

## Структура рантайма

```
<ROOT_DIR>/
├── logs/      ← syncmaster-YYYY-MM-DD.log, audit.json
└── report/    ← report_DDMMYYYY_HHMM.txt
```

`ROOT_DIR` определяется автоматически: в режиме разработки — корень репозитория, в продакшене — директория бандла.
